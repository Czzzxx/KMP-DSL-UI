它通过 JSON 定义界面，在 Android 端使用 Compose 实时渲染。代码结构清晰地划分为：跨平台逻辑层 (shared) 和 平台渲染实现层 (app)。
以下是详细的文件职能说明：
1. 📂 核心逻辑层 (Shared Module)
这部分代码位于 shared/src/commonMain，是整个 DSL 引擎的“大脑”，不依赖 Android 框架。
•
shared/src/commonMain/kotlin/com/example/capcut_project/ui/model/UiNode.kt: UI 树的根基。定义了 UiNode 基类及其子类（Box, Column, Text 等），决定了 DSL 哪些组件可用。
•
shared/src/commonMain/kotlin/com/example/capcut_project/ui/UiNodeSerializer.kt: 多态解析器。负责将 JSON 文本转换成 UiNode 对象。如果遇到不认识的组件，它会转化为 UnknownNode 以防止崩溃。
•
shared/src/commonMain/kotlin/com/example/capcut_project/engine/ExpressionEvaluator.kt: 逻辑解释器。支持 {user.is_new_user} 这种动态表达式，负责在运行时替换数据。
•
shared/src/commonMain/kotlin/com/example/capcut_project/ui/UiValidator.kt: 质检员。在渲染前静态扫描 DSL 里的错误（如颜色格式不对、枚举值非法），并返回错误清单。
•
shared/src/commonMain/kotlin/com/example/capcut_project/ui/model/PropsMapper.kt: 属性转换器。辅助将 JSON 里的 props 字典映射为具体的 Kotlin 字段。
2. 📂 Android 渲染层 (App Module)
这部分代码位于 app/src/main/java/com/example/capcut_project/，负责把“大脑”发出的指令变成手机屏幕上的像素。
核心渲染器 (ui/renderer/)
•
DynamicRenderer.kt: 渲染枢纽。这是最核心的文件，包含 DynamicRenderer 函数，根据 UiNode 类型派发到具体的 Compose 组件（如 Text, Button）。
•
AnimationRenderer.kt: 动效中心。处理入场动画（fadeIn, scaleIn）和循环动画（跑马灯 marquee、数字滚动 numRolling）。
•
ModifierExt.kt: 样式转换工具。将 DSL 里的 width, height, margin 等属性转换为 Compose 的 Modifier。
•
StyleUtils.kt: 细节处理工具。负责解析颜色、对齐方式、内边距等具体数值。
辅助与组件 (ui/)
•
LocalUiErrors.kt: 报错上下文。利用 CompositionLocal 将 UiValidator 发现的错误隐式传递给每个组件。
•
app/src/main/java/com/example/capcut_project/ui/components/ErrorIndicator.kt: 报错 UI。当 DSL 写错时，在界面组件右上角显示一个黄色的感叹号 !，点击可看详情。
3. 📂 业务入口与逻辑
•
MainActivity.kt: 主舞台。
◦
负责从 assets 加载 JSON。
◦
注入环境数据（如 is_new_user）。
◦
管理页面层级（底层是订阅页，顶层是蒙层和弹窗）。
•
ActionHandler.kt (刚重构出来的): 动作中枢。
◦
解耦了点击后的所有逻辑。
◦
处理如：支付、切换 SVIP/VIP 标签、显示挽留弹窗、关闭页面等业务行为。
4. 📂 DSL 配置 (app/src/main/assets/DSL/)
•
subscription_vip.json / subscription_svip.json: 描述了整个订阅页面的结构。
•
retain_popup_xxx.json: 挽留弹窗的独立定义。
💡 整体流程总结：
1.
MainActivity 发起加载请求。
2.
UiNodeSerializer 将 JSON 转为 UiNode 树。
3.
UiValidator 对树进行体检，发现问题存入 LocalUiErrors。
4.
DynamicRenderer 开始递归遍历树，应用 ModifierExt 的样式。
5.
如果节点有动画，由 AnimationRenderer 包装。
6.
用户点击组件，触发 ActionHandler 执行具体的 Android 业务逻辑。
