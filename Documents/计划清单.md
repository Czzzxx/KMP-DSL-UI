### 🏁 第一阶段：基建与解析闭环

**目标：** 确定边界，完成 DSL 到 Kotlin 强类型模型（IR）的转换。

- 敲定场景与数据结构**
  - 从“剪映订阅挽留弹窗”作为靶向场景。
  - 定义最初版的 JSON DSL 结构（包含 type, styles, children, props）。
  - **产出：** 一份静态的 JSON 样例草稿。
- KMP 工程初始化与解析器 (Parser)**
  - 搭建 KMP 工程，主攻 commonMain。
  - 使用 data class 和 sealed class 建立 UI 节点树模型。
  - *提示：这部分逻辑与前端组件化框架的数据驱动思路非常一致，利用好这层直觉，把 JSON 安全地反序列化为 Kotlin 模型。*
- 容错与校验**
  - 编写校验逻辑，拦截诸如“缺失必填字段”、“颜色格式非法”、“未知组件类型”等异常。
  - **产出：** 能够在控制台打印出正确解析的树结构，或准确抛出自定义错误的 Parser。
